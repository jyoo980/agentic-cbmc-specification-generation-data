`run-cbmc` runs CBMC with a fixed `--depth 200` (`_CBMC_DEPTH` in `tools/run_cbmc_and_mutation_testing.py`, not env-configurable; do not hard-code depth in specs).

In `Syzygy_Zopfli/c_code/zopfli.c` (which `#include`s `<stdlib.h>`), each `__CPROVER_is_fresh(...)` in a `requires` clause routes through CBMC's full `malloc` model (memory-map disjointness, leak tracking, should-fail check) — ~70 GOTO steps per fresh object. A function needing valid memory for ~4 pointer params (e.g. `AddBit(int bit, unsigned char *bp, unsigned char **out, size_t *outsize)`) builds a harness of ~280–300 steps, which EXCEEDS depth 200 before the body runs.

Effect: CBMC truncates the path, the postconditions (checked at function exit) are never reached, so they pass vacuously. The tool reports "verified successfully" but it is a vacuous proof — and mutation testing then kills 0 mutants regardless of how strong the spec is. Symptom to recognize: adding `__CPROVER_ensures(1 == 0)` still "verifies" (run via direct cbmc and grep `postcondition.*SUCCESS`; raise `--depth` to ~350 to see it flip to FAILURE).

Without `<stdlib.h>` the same 4-is_fresh harness fits in depth 200 (light built-in alloc) — so it's the malloc model, not is_fresh per se. `rw_ok` for scalars is lighter but unsound for written-through out-params. No sound formulation fits depth 200 here.

Confirmed instances: `AddBit`, `AddLZ77Data`, `BoundaryPMFinal`, `BoundaryPM`, and `ExtractBitLengths` (recursive; params `Node*(*lists)[2], leaves, pool` — its self-call + `InitNode` are both replaced by contracts, so the contract must be inductive). For `BoundaryPM` a requires(is_fresh pool/lists/leaves)+assigns(object_whole(lists),pool->next) contract verifies at depth 200 but is vacuous (ensures(1==0) SUCCEEDS at 200, FAILS at 400). Extra obstacle: any `__CPROVER_ensures` that dereferences a nested list pointer (e.g. `__CPROVER_old(lists[index][1]->count)`) fails NON-vacuously — the entry-time snapshot derefs the nondeterministic `lists[i][j]` (R_OK/bounds FAILURE at the body's first `lists[index][1]->count`). Only scalar facts like `pool->next >= __CPROVER_old(pool->next)` are safely addable. Net: kill score 0/27 is tooling-forced, not spec weakness. `BoundaryPMFinal` (same `Node*(*lists)[2],leaves,pool` shape, plus nested `is_fresh` on `lists[index][1]`, `lists[index-1][0/1]`, `pool->next` so even heavier harness) verifies a STRONG branch-conditional `ensures` (deref nested list pointers — safe here because those nested pointers are made fresh in requires), but `ensures(0==1)` still SUCCEEDS at depth 200 AND 450 → vacuous, 0/16 kills forced.

`ExtractBitLengths(Node *chain, Node *leaves, unsigned *bitlengths)` (3 indexed pointer params; restricted to single-node chain via `requires chain->tail==0` so the `chain->tail` walk is safe without per-node freshness) directly demonstrates the wall both ways: (1) `is_fresh` on all 3 → "verified" but VACUOUS (0/12 kills, loop-body mutants at lines 332-340 all survive). (2) Swapping the two *indexed* pointers (`leaves`, `bitlengths`) to the light `r_ok`/`w_ok` predicates DOES fit depth 200 and REACHES the body (non-vacuous) — but then CBMC cannot discharge the array bounds / `assigns`-assignable checks: `r_ok`/`w_ok`, even combined with `__CPROVER_OBJECT_SIZE(p)==n` and `__CPROVER_POINTER_OFFSET(p)==0` assumptions, do NOT make the bounds checker treat the object as sized — `bitlengths[leaves[k].count]` reports "pointer outside object bounds" + "is assignable: FAILURE". Only `is_fresh` establishes a bounded object for write-through/indexed accesses, and only `is_fresh` is heavy enough to overrun depth 200. So: indexed pointers MUST be `is_fresh` (sound) → vacuous; or light predicates (fits) → unsound bounds FAILURE. No middle ground. Net 0/12 is tooling-forced.

`AddLZ77Data(lz77, lstart, lend, expected_data_size, ll_symbols, ll_lengths, d_symbols, d_lengths, bp, out, outsize)` is the heaviest seen: ~11 pointers needing memory (`lz77` + its `litlens`/`dists` arrays, four `ZOPFLI_NUM_LL`/`ZOPFLI_NUM_D` tables, `bp`, `outsize`, `out`, `*out`). A sound strong contract (is_fresh on all, forall bounding litlens/dists per literal-vs-match, ll/d_lengths in [1,15], expected_data_size==0) verifies at depth 200 but 0/40 kills — even `i<lend`→`i>lend` and the literal/match `dist==0`→`dist!=0` branch mutants survive, proving the loop body is never reached. Compounding the vacuity, it is also non-vacuously UNVERIFIABLE if reached: the callees `AddHuffmanBits`/`AddBits` are contract-replaced and their `ensures` constrain only `*bp`/`*outsize` (not buffer validity), so after the first call `*out` is havoc'd and the second sequential call's `requires is_fresh(*out, *outsize+1)` cannot be discharged. So lightening the harness to fit depth 200 would flip it to FAILURE, not kills. Net 0/40 tooling-forced.

`AddNonCompressedBlock(options, final, in, instart, inend, bp, out, outsize)` (4 pointer params needing memory: `in`, `bp`, `out`, `*out`, `outsize`; `options` is only `(void)`-cast so never deref'd, no is_fresh needed) is the same wall. A general contract (is_fresh all, `*bp<=7`, `*outsize` in [1, max_malloc/2], `inend-instart<=8`, ensures `*bp==0` and `*outsize>=old+4`) verifies at depth 200 → 0/26 kills. I also tried the OPPOSITE extreme to force reachability: minimal harness `*outsize==1` + `*bp==0` + `inend-instart<=1` (so the in-body `ZOPFLI_APPEND_DATA` memset stays 1-2 bytes, ruling out "large memset blows depth" as the cause) PLUS byte-precise postconditions pinning every emitted byte ((*out)[1]=BFINAL, [2..5]=LEN/NLEN header, [6]=data). Still 0/26 — so it is the is_fresh/malloc *harness setup* exhausting depth 200, not the loop body's memset size. No spec change rescues kills here.

`CalculateBlockSymbolSizeSmall(ll_lengths, d_lengths, lz77, lstart, lend)` (same lz77+litlens/dists+ll/d_lengths shape as `AddLZ77Data`, 4 indexed pointers + lz77 struct deref = 5 is_fresh objects) is the same wall. A sound strong contract (is_fresh on all 5, `lend<=lz77->size`, forall bounding `litlens[i]<=258` and `dists[i]!=0 ==> litlens[i]>=3 && dists[i]<=32768` for callee preconds, `assigns()`, ensures `ret>=ll_lengths[256]` and `lstart==lend ==> ret==ll_lengths[256]`) verifies at depth 200 → 0/16 kills, all 16 surviving mutants are loop-guard/body RELATIONAL ops (never reached). Confirmed vacuous: added `ensures(1==0)` → SUCCESS at depth 200, but postcondition.3 FAILS at depth 600 (body finally reached). Net 0/16 tooling-forced; left the strong sound spec in place.

`CalculateBlockSymbolSizeGivenCounts(ll_counts, d_counts, ll_lengths, d_lengths, lz77, lstart, lend)` is the HEAVIEST count-table case: its `if` branch delegates to `CalculateBlockSymbolSizeSmall` (so needs the full lz77+litlens+dists+forall-bounds harness) AND its `else` branch sums over four full histogram tables (so needs is_fresh on ll_counts/d_counts/ll_lengths/d_lengths too) = 7 is_fresh objects. A sound strong contract (is_fresh all 7, lstart<=lend<=lz77->size, forall litlens<=258 & dists DEFLATE bounds, assigns(), ensures ret>=ll_lengths[256]) verifies at depth 200 → 0/21 kills, all 21 surviving mutants are the if-guard relational/arith ops at line 317 and the loop bodies (never reached). Confirmed vacuous via the documented `ensures(1==0)` trick: it still "verifies" (mutation testing only runs post-success). Net 0/21 tooling-forced; left the strong sound spec in place.

`TryOptimizeHuffmanForRle(lz77, lstart, lend, ll_counts, d_counts, ll_lengths, d_lengths)` is the same wall — it calls `CalculateTreeSize`, `CalculateBlockSymbolSizeGivenCounts` (twice), `OptimizeHuffmanForRle`, `ZopfliCalculateBitLengths`, `PatchDistanceCodesForBuggyDecoders`, all contract-replaced, so its requires is the UNION of theirs: is_fresh on lz77 + nested litlens/dists + ll_counts/d_counts/ll_lengths/d_lengths (7 is_fresh objects), forall litlens<=258 & DEFLATE dist bounds, ll/d_lengths<=15, counts<2^(64-9), assigns(object_whole(ll_lengths),object_whole(d_lengths)), ensures ret>=26. Verifies at depth 200 → 0/9 kills; all 9 survivors are the line-1272 min-select comparison and the two `treesize+datasize` return sums (never reached). Confirmed vacuous via `ensures(ret>=100000000)` → still SUCCESS at depth 200. Two extra gotchas while building it: (1) the quantified `requires` deref `litlens[qi]` generates a well-definedness BOUNDS assertion at the *caller* even though it's only assumed in the callee — guard the quantifier with `qi < lz77->size`; (2) unconstrained `lz77->size` makes the is_fresh byte-count `size * sizeof(*litlens)` overflow size_t (counterexample: size≈2^63), so add `requires(lz77->size <= ((size_t)-1)/sizeof(*lz77->litlens))`. Net 0/9 tooling-forced; left the strong sound spec in place.

`ZopfliLZ77GetHistogramAt(lz77, lpos, ll_counts, d_counts)` is the same wall — it needs 8 is_fresh objects: `lz77` struct + its nested `ll_counts`/`d_counts`/`ll_symbol`/`d_symbol`/`dists` arrays + the two output tables `ll_counts`/`d_counts`. A sound strong contract verifies at depth 200 → 0/41 kills, all 41 survivors are the cumulative-index arithmetic (`llpos+i`, `lpos/ZOPFLI_NUM_LL`) and the decrement loops (never reached). Confirmed vacuous via the `ensures(1==0)` trick → still SUCCESS at depth 200. Two sizing details worth keeping: the cumulative arrays are indexed only up to the end of lpos's chunk, so size them exactly `(ZOPFLI_NUM_LL*(lpos/ZOPFLI_NUM_LL)+ZOPFLI_NUM_LL)` / `(ZOPFLI_NUM_D*(lpos/ZOPFLI_NUM_D)+ZOPFLI_NUM_D)` elements; and the symbol arrays are read at indices `lz77->ll_symbol[i]`/`d_symbol[i]` used to index the output tables, so a `forall (lpos<qi<size) ==> symbol[qi] < ZOPFLI_NUM_*` requires-bound is needed (guard the quantifier with `qi<size`). Net 0/41 tooling-forced; left the strong sound spec in place.

`ZopfliLZ77GetHistogram(lz77, lstart, lend, ll_counts, d_counts)` (the caller of `ZopfliLZ77GetHistogramAt`) is the same wall — 8 is_fresh objects (lz77 struct + nested ll_counts/d_counts/ll_symbol/d_symbol/dists + the two output tables). Its small-range branch counts symbols directly; its large-range branch calls the contract-replaced `ZopfliLZ77GetHistogramAt` (whose contract has NO ensures, so the output is fully havoc'd → no output postcondition can hold there anyway). Sound strong contract: is_fresh all 8 (size the cumulative arrays `ZOPFLI_NUM_LL*(size/ZOPFLI_NUM_LL)+ZOPFLI_NUM_LL` so they cover every lpos<=size-1 without size-1 underflow when size=0), `lstart<=lend<=lz77->size`, overflow guard `size <= ((size_t)-1)/sizeof(*ll_counts) - ZOPFLI_NUM_LL`, `forall qi<size ==> ll_symbol[qi]<ZOPFLI_NUM_LL` and same for d_symbol (full-range, since the callee needs (lpos,size)), assigns(object_whole(ll_counts/d_counts)). Verifies at depth 200 → 0/29 kills; all 29 survivors are the branch-guard (line 1371), the direct-count loop guard/dist-test (1375/1378), the `lstart>0` guard (1387), the subtraction-loop guards (1393/1397) and the `lend-1`/`lstart-1` callee args (1386/1391). Confirmed vacuous via the documented probe: `ensures((lstart+ZOPFLI_NUM_LL*3>lend) ==> d_counts[0]==999999)` (a false branch-1 claim) still SUCCEEDS at depth 200. Also note loop contracts are USELESS here: the avocado pipeline runs `goto-instrument --partial-loops --unwind 5` WITHOUT `--apply-loop-contracts`, so `__CPROVER_loop_invariant`/loop-`assigns` annotations are silently ignored (loops are just unrolled 5x). Net 0/29 tooling-forced; left the strong sound spec in place. See also [[cbmc-partialloops-unwind5-truncation]].

`GetDynamicLengths(lz77, lstart, lend, ll_lengths, d_lengths)` is a straight-line wrapper (ZopfliLZ77GetHistogram → ZopfliCalculateBitLengths x2 → PatchDistanceCodesForBuggyDecoders → TryOptimizeHuffmanForRle, all contract-replaced; ll_counts/d_counts are in-frame locals). Wrote the union contract: is_fresh lz77 + nested litlens/dists/ll_symbol/d_symbol/ll_counts/d_counts (cumulative arrays sized ZOPFLI_NUM_**(size/ZOPFLI_NUM_*)+ZOPFLI_NUM_*) + is_fresh ll_lengths/d_lengths, histogram overflow guard, symbol<ZOPFLI_NUM_* and litlens<=258/dist-DEFLATE foralls (litlens/dists foralls guarded qa<size to avoid caller well-definedness bounds assertion), assigns(object_whole ll_lengths/d_lengths), ensures ret>=26. Verifies at depth 200. Note: avocado reports "Mutation testing not possible; no mutable operators" — the body has no mutable operators (only sequential calls + `ll_counts[256]=1`), so kill score is N/A here regardless of vacuity, not a spec-strength signal.

`GetFixedTree(unsigned *ll_lengths, unsigned *d_lengths)` (fills the fixed-Huffman tables: ll_lengths[0..287], d_lengths[0..31] via 5 constant-bound loops; NO callees, NO struct/nested pointers — just 2 is_fresh objects) is the LIGHTEST-param case that STILL hits the wall. Sound strong contract: is_fresh(ll_lengths, ZOPFLI_NUM_LL*sizeof) + is_fresh(d_lengths, ZOPFLI_NUM_D*sizeof), assigns(object_whole(ll_lengths), object_whole(d_lengths)), 5 full-range `__CPROVER_ensures(__CPROVER_forall{...})` pinning every element's exact value (8/9/7/8 for the ll ranges, 5 for d). Verifies at depth 200 → 0/25 kills; all 25 survivors are the loop-guard RELATIONAL mutants. Confirmed vacuous: postconditions SUCCEED at depth 200/250/300/350/400/500 but FAIL at depth 1000 (the 2 large is_fresh malloc harnesses + 25 unrolled loop iterations need >500 GOTO steps to reach the function-exit postcondition; at depth 200 it's unreachable → vacuous). Note the full-range foralls are sound w.r.t. real C but FALSE under `--partial-loops --unwind 5` (loops run only 5 iters), which is why they flip to FAILURE once reachable — but at depth 200 they pass vacuously so avocado succeeds. Spec content can't help: it doesn't move WHERE the postcondition is checked (function exit), so no spec is reachable at depth 200. is_fresh is mandatory (function writes ll_lengths[i]/d_lengths[i] up to index 287/31; w_ok → bounds FAILURE). Two GOTCHAS while writing it: (1) forall quantifier var names must be UNIQUE across all ensures clauses in one function or goto-cc errors `redeclaration of '...::ensures::N::q' with no linkage` — use qa/qb/qc/... not the same `qi` everywhere. (2) loop contracts here trip the tree-sitter mutation parser: `for(...) __CPROVER_assigns(...) __CPROVER_loop_invariant(forall{...}) stmt;` with the forall-brace clause LAST before the body makes the whole function vanish from the mutant generator ("Function 'GetFixedTree' missing from file"); reorder so a simple `__CPROVER_loop_invariant(i <= N)` sits last before the body (see [[cbmc-treesitter-loopcontract-misparse]]). Since loop contracts are ignored by the pipeline anyway, I removed them and kept only the function-level contract. Net 0/25 tooling-forced; left the strong sound spec in place.

`ZopfliLZ77GetByteRange(lz77, lstart, lend)` is a one-line PER-PATH vacuity case worth noting: it needs 4 is_fresh objects (lz77 struct + nested `pos`/`dists`/`litlens` — all three indexed at `lend-1` and `lstart`). Sound strong contract: is_fresh all 4, `lstart<=lend<=lz77->size`, overflow guard `size<=((size_t)-1)/sizeof(*lz77->pos)`, branch-conditional ensures (`lstart==lend ==> ret==0`; else `ret == pos[lend-1] + (dists[lend-1]==0?1:litlens[lend-1]) - pos[lstart]`, exactly mirroring the body). Verifies at depth 200 → 0/5 kills (all 5 survivors are the line-1515/1512 arith/relational ops on the dereference path). PARTIAL vacuity confirmed: `ensures(1==0)` FAILS at depth 200 (the trivial `lstart==lend` return-0 path IS reached), but `ensures(lstart!=lend ==> 1==0)` SUCCEEDS at depth 200 — the array-deref path (where every mutant lives) is truncated before the postcondition. So the cheap path reaches exit while the 4-is_fresh-deref path overruns depth 200. Net 0/5 tooling-forced; left strong sound spec in place. Lesson: when `ensures(1==0)` fails but kills are still 0, probe each branch separately — a trivial early-return path can mask vacuity on the heavy path.

`AddLZ77Block(options, btype, final, lz77, lstart, lend, expected_data_size, bp, out, outsize)` is the same wall — the heaviest pointer count yet: `options` + `lz77` struct + nested `litlens`/`dists`/`pos`/`data` + `bp` + `outsize` + `out` + `*out` = ~10 is_fresh objects. btype==2 (dynamic) is structurally out of scope (calls `AddDynamicTree` whose contract requires `out==NULL`, incompatible with the real non-null double-pointer); btype==0 (stored) is coverable by pinning a single literal command (`lend==lstart+1`, `dists[lstart]==0`, `pos[lstart]==0`, is_fresh(data,1)) so `ZopfliLZ77GetByteRange` returns 1 and `AddNonCompressedBlock`'s range<=8 / is_fresh(data,end=1) hold. Wrote a sound strong contract for btype∈{0,1} (unconditional is_fresh on all arrays sized by lz77->size with lend<=size — NOT conditional is_fresh, which makes the guarded path vacuous/unreachable; value constraints can be btype-guarded safely). Verifies at depth 200 → 0/14 kills. Confirmed vacuous via `ensures(*bp==99)` → still SUCCESS. The btype==1-only variant (8 is_fresh) was equally vacuous, so dropping the stored path doesn't help. Also note: 9 of the 14 mutants (lines 2128-2151, the `uncompressed_size`/`compressed_size` loop and `/1024` arith) only feed `fprintf` diagnostics and are UNOBSERVABLE through any contract regardless of vacuity — even a reachable spec couldn't kill them. AddLZ77Data (a callee) has NO ensures, so it havocs *bp/*outsize; only the trailing end-symbol AddHuffmanBits pins `*bp<=7`, the lone provable postcondition. Net 0/14 tooling-forced; left the strong sound spec in place.

`ZopfliCalculateBlockSize(lz77, lstart, lend, btype)` is the same wall — its three btype branches union to need `lz77` struct + nested `litlens`/`dists`/`pos`/`ll_symbol`/`d_symbol`/`ll_counts`/`d_counts` = ~8 is_fresh objects (the heaviest seen via a single struct param; `ll_lengths`/`d_lengths` are in-frame locals). Sound strong contract: is_fresh all, `lstart<=lend<=size`, histogram overflow guard `size <= ((size_t)-1)/sizeof(*ll_counts) - ZOPFLI_NUM_LL`, symbol<ZOPFLI_NUM_* foralls, litlens<=258/DEFLATE-dist foralls (guarded qa<size), assigns(), btype-conditional ensures (`btype==0 ==> ret>=0`; `btype==1 ==> ret>=10` since GetFixedTree pins ll_lengths[256]==7 + 3-bit header; `btype not in {0,1} ==> ret>=29` since GetDynamicLengths>=26 + header). Verifies at depth 200 → 0/6 kills; all 6 survivors are in the btype==0 uncompressed branch (the `blocks*5*8 + length*8` / `length/65535` / `length%65535` arith at lines 1638-1643) plus the two btype branch-selectors (lines 1635/1645) — never reached. Confirmed vacuous via `ensures(1==0)` → still SUCCESS at depth 200. Net 0/6 tooling-forced; left strong sound spec in place.

`ZopfliInitCache(size_t blocksize, ZopfliLongestMatchCache *lmc)` is the same wall but proves IN-BODY malloc also loads the depth harness: only ONE is_fresh param (`lmc`), yet it `malloc`s 3 arrays (`length`/`dist`/`sublen`) in-body — those 3 malloc-model setups + reaching the function-exit postcondition past the loop-fills exceed depth 200. Sound strong contract: is_fresh(lmc), `blocksize>=1`, overflow guard `blocksize <= __CPROVER_max_malloc_size/(ZOPFLI_CACHE_LENGTH*3)`, assigns(lmc->length,lmc->dist,lmc->sublen), 3 full-range `ensures(forall ...)` pinning length[i]==1/dist[i]==0/sublen[i]==0. Verifies at depth 200 → 0/16 kills, all 16 survivors are the 3 loop-guard RELATIONAL mutants (lines 3202/3207/3212) + the NULL-check `==`->`!=`. Confirmed vacuous via `ensures(1==0)`: SUCCESS at depth 200, FAILURE at depth 1200 (body reached; real foralls also FAIL there since `--partial-loops --unwind 5` truncates the loops). Parse gotcha (same as GetFixedTree): adding `__CPROVER_loop_invariant` loop contracts made ZopfliInitCache vanish from the mutant generator ("Function missing from file") — and loop contracts are ignored by the pipeline anyway, so I removed them and kept only the function-level forall ensures. Note function-contract forall ensures parse FINE (GetFixedTree proves it); it's the LOOP-contract forall that breaks tree-sitter. Net 0/16 tooling-forced; left the strong sound spec in place.

`StoreInLongestMatchCache(s, pos, limit, sublen, distance, length)` is the same wall — needs 6 is_fresh objects: `s` struct + nested `s->lmc` struct + `s->lmc->length`/`s->lmc->dist`/`s->lmc->sublen` arrays + the `sublen` param array. Sound strong contract: is_fresh all 6, `pos>=s->blockstart`, lmcpos overflow guard `pos-blockstart < max_malloc/(ZOPFLI_CACHE_LENGTH*3)`, `length<=ZOPFLI_MAX_MATCH`, sublen sized `(length+1)`, and the cache-init invariant `s->lmc->length[lmcpos]==1` (needed so the in-body `assert(length[lmcpos]==1 && dist[lmcpos]==0)` holds), assigns(dist[lmcpos], length[lmcpos], object_whole(sublen)), branch-conditional ensures keyed on `limit==ZOPFLI_MAX_MATCH && old(dist[lmcpos])==0` pinning length[lmcpos]/dist[lmcpos] to `length<3?0:length` / `length<3?0:distance` (and unchanged otherwise). Verifies at depth 200 → 0/25 kills; all 25 survivors are the cache_available/branch-guard relational+logical ops (lines 2357-2365) — never reached. Confirmed vacuous via `ensures(1==0)` → still SUCCESS at depth 200. Net 0/25 tooling-forced; left the strong sound spec in place.

`ZopfliCacheToSublen(lmc, pos, length, sublen)` is the same wall — 3 is_fresh objects (`lmc` struct + nested `lmc->sublen` array + the `sublen` output array) plus a contract-replaced callee (`ZopfliMaxCachedSublen`) and two unrolled loops.
